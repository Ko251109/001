# プロンプトの基礎

## はじめに

AIに話しかけるとき、何を・どう伝えるかで結果がガラリと変わることって、ありませんか？

その「AIへの伝え方」のことを **プロンプト** といいます。プロンプトがうまくなると、AIがぐっと使いやすくなります。この記事では、そのコツをやさしく解説していきます！

---

## 1. プロンプトって何？

プロンプトは、AIへの「お願いのメモ」みたいなものです。AIにやってほしいことを言葉で伝えるための入力テキストで、こんな要素を組み合わせて作ります。

- **指示（Instruction）**: 「〜してください」という頼みごと
- **文脈（Context）**: 「こんな状況なんです」という背景説明
- **入力データ（Input Data）**: 処理してほしいテキストやデータ
- **出力形式（Output Indicator）**: 「こんな感じで返してね」という形式の指定

全部そろっていなくても大丈夫。でも、多いほどAIは助かります！

---

## 2. プロンプトの基本構造

うまいプロンプトには、こんな構造があります。

```
[役割の設定] + [文脈・背景] + [具体的な指示] + [出力形式の指定]
```

百聞は一見にしかず！具体例を見てみましょう。

### ちょっと惜しいプロンプト

```
メールを書いて
```

AIも「え、どんなメール？」ってなっちゃいます。

### グッとよくなったプロンプト

```
あなたはビジネスメールのプロフェッショナルです。
以下の状況に基づいて、丁寧でプロフェッショナルな謝罪メールを書いてください。

状況: 納品が2日遅延した
相手: 取引先の田中部長
目的: 謝罪と今後の対応策の説明

箇条書きは使わず、300文字以内で書いてください。
```

情報がそろうと、AIもびしっと答えてくれます！

## 3. 効果的なプロンプトの4つの原則

### 3.1 明確に伝えよう（Clarity）

ぼんやりした指示より、ハッキリした指示の方がずっと良い結果になります。

| ぼんやり例 | ハッキリ例 |
|--------|--------|
| 「説明して」 | 「中学生にもわかるように、100文字以内で説明して」 |
| 「翻訳して」 | 「以下の文章を、ビジネス文書として適切な英語に翻訳して」 |
| 「要約して」 | 「以下の文章を3つの箇条書きで要約して」 |

### 3.2 具体的に指定しよう（Specificity）

出力の形式・長さ・スタイルを教えてあげると、AIはとても喜びます。

```
# こんなふうに具体的に！
- 長さ: 「500文字以内」「3段落で」「箇条書き5点」
- 形式: 「マークダウン形式で」「JSON形式で」「表形式で」
- 対象: 「初心者向けに」「エンジニア向けに」「子供向けに」
- トーン: 「丁寧に」「カジュアルに」「フォーマルに」
```

### 3.3 背景を教えよう（Context）

AIはあなたの状況を知らないので、背景情報はしっかり伝えましょう。

```
# 背景なし（AIが困る例）
このコードのバグを直して

# 背景あり（AIが助かる例）
以下はPython 3.10で書かれたWebスクレイピングのコードです。
実行するとIndexErrorが発生します。
エラーの原因を特定し、修正したコードを提示してください。

[コードをここに貼り付け]
```

### 3.4 役割を与えよう（Role Prompting）

「あなたは〇〇です」と役割を設定すると、AIの回答がぐっと専門的になります！

```
あなたは10年以上の経験を持つシニアソフトウェアエンジニアです。
コードレビューの観点から、以下のコードの問題点を指摘してください。
```

## 4. プロンプトの種類、3つ覚えよう

### 4.1 Zero-shot プロンプト

例なしでシンプルにお願いするスタイルです。

```
次の文章をポジティブ・ネガティブ・中立に分類してください:
「今日の会議はいつもより長かった」
```

### 4.2 Few-shot プロンプト

「こんな感じで！」と例を見せてからお願いする方法です。精度がアップします。

```
以下の例を参考に、文章を分類してください:

例1: 「素晴らしいサービスでした！」 → ポジティブ
例2: 「最悪な経験でした」 → ネガティブ
例3: 「普通でした」 → 中立

分類対象: 「まあまあ良かったと思います」
```

### 4.3 Chain-of-Thought（CoT）プロンプト

「ステップバイステップで考えて！」と伝えると、難しい問題でも精度が上がります。

```
以下の問題をステップバイステップで解いてください。
各ステップの考え方を説明しながら答えを導いてください。

問題: [複雑な数学や論理問題]
```

## 5. やりがちな失敗、チェックしてみよう

### 一度に詰め込みすぎ

「あれもこれも」と盛り込みすぎると、AIも混乱します。複数のプロンプトに分けるとスッキリします。

### 否定形ばかり使う

「〜しないで」は意外とうまく伝わりにくいです。ポジティブな言い方の方が効果的！

| 否定形（ちょっと惜しい） | 肯定形（おすすめ） |
|----------------|--------------|
| 「専門用語を使わないで」 | 「中学生でも理解できる言葉で」 |
| 「長くしないで」 | 「100文字以内で」 |

### 背景情報が足りない

AIはあなたの状況を何も知りません。「言わなくてもわかるよね？」は通じないので、必要な情報はしっかり書きましょう！

---

## 6. もっとうまくなるベストプラクティス

1. **試して改善する**: 最初から完璧なプロンプトは誰も書けません。結果を見て少しずつ磨いていきましょう
2. **テンプレートを作る**: よく使うプロンプトはメモしておくと便利です
3. **区切り記号を使う**: 指示とデータは `---` や バッククォートで区別すると、AIが読み取りやすくなります
4. **出力例を見せる**: 「こんな形式で！」と例を添えると、イメージが伝わりやすくなります
5. **制約を伝える**: 「〇〇は含めないで」「必ず〇〇を入れて」などの条件もしっかり書きましょう

---

## 7. システムプロンプトとユーザープロンプト

APIを使う場合、プロンプトは2種類に分かれます。

| 種類 | 役割 | 例 |
|------|------|----|
| **システムプロンプト** | AIの基本的な振る舞いや役割を定義する土台 | 「あなたは日本語のみで回答するアシスタントです」 |
| **ユーザープロンプト** | 個別の質問や指示 | 「Pythonでソートアルゴリズムを実装してください」 |

システムプロンプトはいわば「キャラクター設定」、ユーザープロンプトは「その日のお願い」というイメージです！

---

## まとめ

プロンプトのコツ、ざっとまとめるとこんな感じです。

- **ハッキリ伝える**: 何をしてほしいかを具体的に書く
- **背景を教える**: 自分の状況や前提をしっかり伝える
- **役割を与える**: 「あなたは〇〇です」で出力の質がアップする
- **形式を指定する**: 長さや形式まで書くと完璧
- **繰り返し改善する**: 一発で決めようとしないことが大切

プロンプトは「AIとの会話の技術」です。使えば使うほど上手になるので、ぜひいろいろ試してみてください！きっと「こんなに変わるの！？」という体験ができますよ。
