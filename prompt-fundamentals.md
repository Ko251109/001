# プロンプトの基礎

## はじめに

プロンプト（Prompt）とは、AIモデルに対して与える入力テキストのことです。プロンプトの書き方によって、AIの出力品質は大きく変わります。この記事では、効果的なプロンプトを書くための基礎知識を解説します。

---

## 1. プロンプトとは何か

プロンプトは、AIモデルへの「指示書」です。人間がAIに何をしてほしいかを伝えるための手段であり、以下の要素を含むことができます。

- **指示（Instruction）**: AIに何をしてほしいかの命令
- **文脈（Context）**: AIが回答するために必要な背景情報
- **入力データ（Input Data）**: 処理対象のテキストやデータ
- **出力形式（Output Indicator）**: 期待する出力の形式や長さ

---

## 2. プロンプトの基本構造

効果的なプロンプトは、以下の構造を持ちます。

```
[役割の設定] + [文脈・背景] + [具体的な指示] + [出力形式の指定]
```

### 例：悪いプロンプト

```
メールを書いて
```

### 例：良いプロンプト

```
あなたはビジネスメールのプロフェッショナルです。
以下の状況に基づいて、丁寧でプロフェッショナルな謝罪メールを書いてください。

状況: 納品が2日遅延した
相手: 取引先の田中部長
目的: 謝罪と今後の対応策の説明

箇条書きは使わず、300文字以内で書いてください。
```

---

## 3. 効果的なプロンプトの原則

### 3.1 明確性（Clarity）

曖昧な表現を避け、具体的に指示します。

| 悪い例 | 良い例 |
|--------|--------|
| 「説明して」 | 「中学生にもわかるように、100文字以内で説明して」 |
| 「翻訳して」 | 「以下の文章を、ビジネス文書として適切な英語に翻訳して」 |
| 「要約して」 | 「以下の文章を3つの箇条書きで要約して」 |

### 3.2 具体性（Specificity）

出力の形式、長さ、スタイルを明示します。

```
# 具体的な指定例
- 長さ: 「500文字以内」「3段落で」「箇条書き5点」
- 形式: 「マークダウン形式で」「JSON形式で」「表形式で」
- 対象: 「初心者向けに」「エンジニア向けに」「子供向けに」
- トーン: 「丁寧に」「カジュアルに」「フォーマルに」
```

### 3.3 文脈の提供（Context）

AIに必要な背景情報を与えます。

```
# 文脈なし（悪い例）
このコードのバグを直して

# 文脈あり（良い例）
以下はPython 3.10で書かれたWebスクレイピングのコードです。
実行するとIndexErrorが発生します。
エラーの原因を特定し、修正したコードを提示してください。

[コードをここに貼り付け]
```

### 3.4 役割の設定（Role Prompting）

AIに特定の役割を与えることで、出力の品質と一貫性が向上します。

```
あなたは10年以上の経験を持つシニアソフトウェアエンジニアです。
コードレビューの観点から、以下のコードの問題点を指摘してください。
```

---

## 4. プロンプトの種類

### 4.1 Zero-shot プロンプト

例示なしで直接タスクを指示します。

```
次の文章をポジティブ・ネガティブ・中立に分類してください:
「今日の会議はいつもより長かった」
```

### 4.2 Few-shot プロンプト

例を示してからタスクを指示します。

```
以下の例を参考に、文章を分類してください:

例1: 「素晴らしいサービスでした！」 → ポジティブ
例2: 「最悪な経験でした」 → ネガティブ
例3: 「普通でした」 → 中立

分類対象: 「まあまあ良かったと思います」
```

### 4.3 Chain-of-Thought（CoT）プロンプト

AIに思考過程を示させることで、複雑な問題の精度が上がります。

```
以下の問題をステップバイステップで解いてください。
各ステップの考え方を説明しながら答えを導いてください。

問題: [複雑な数学や論理問題]
```

---

## 5. よくある失敗パターン

### 5.1 過度に複雑な指示

一度に多くのことを求めすぎると、品質が落ちます。複数のプロンプトに分割しましょう。

### 5.2 否定形の多用

「〜しないで」より「〜してください」の方が効果的です。

| 否定形（避ける） | 肯定形（推奨） |
|----------------|--------------|
| 「専門用語を使わないで」 | 「中学生でも理解できる言葉で」 |
| 「長くしないで」 | 「100文字以内で」 |

### 5.3 文脈の不足

AIはあなたの状況を知りません。必要な情報は必ず明示しましょう。

---

## 6. プロンプトエンジニアリングのベストプラクティス

1. **反復改善**: 最初から完璧なプロンプトは書けません。出力を見て改善を繰り返す
2. **テンプレート化**: よく使うプロンプトはテンプレートとして保存する
3. **区切り記号の活用**: 指示とデータを明確に区別する（`---`、`###`、バッククォートなど）
4. **出力例の提示**: 「以下のような形式で出力してください」と例を見せる
5. **制約の明示**: 禁止事項や条件を明確に伝える

---

## 7. システムプロンプトとユーザープロンプト

APIを使う場合、プロンプトは2種類に分かれます。

| 種類 | 役割 | 例 |
|------|------|----|
| **システムプロンプト** | AIの基本的な振る舞いや役割を定義 | 「あなたは日本語のみで回答するアシスタントです」 |
| **ユーザープロンプト** | 個別の質問や指示 | 「Pythonでソートアルゴリズムを実装してください」 |

---

## まとめ

効果的なプロンプトを書くためのポイントをまとめます。

- **明確に**: 何をしてほしいかを具体的に伝える
- **文脈を与える**: 必要な背景情報を提供する
- **役割を設定する**: AIに適切な役割を与える
- **形式を指定する**: 出力の形式や長さを明示する
- **反復する**: 出力を見て継続的に改善する

プロンプトエンジニアリングは、AIをより効果的に活用するための重要なスキルです。日々の実践を通じて、より良いプロンプトを書けるようになりましょう。
