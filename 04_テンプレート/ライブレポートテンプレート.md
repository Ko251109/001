# ライブレポートテンプレート

> このテンプレートをベースに、`03_表現辞典/頻出表現辞典.md` の表現を使って記事を書く。

---

## テンプレートA：標準ライブレポート

```markdown
[アーティスト名]のライブに行ってきました。

[背景・文脈：なぜ行ったか、何回目か、前回との比較など 1〜2文]

今回の公演は[会場名]にて。[会場の特徴や規模感 1文]

オープニングは「[曲名]」から。[序盤の流れ・雰囲気 2〜3文]

序盤は[セットリストの流れ]など、[特徴的な楽曲群の説明]が続きます。

[中盤の描写。MCがあれば「MCでは、〜の話題もありました。」と挿入]

終盤は[クライマックスの描写]。会場の熱量が一段階上がったのをはっきりと感じました。

アンコールでは[アンコールの内容]。

[締め：余韻・感想・次への期待 2〜3文]
```

---

## テンプレートB：フェスレポート

```markdown
今年も[フェス名]に行ってきました。[参加日程・規模感]

[観たアーティスト1]

[観たアーティスト2]

[観たアーティスト3]

[全体の感想・まとめ]
```

---

## テンプレートC：年間ベスト記事

```markdown
色々掲載場所を変えながら毎年勝手にやっています。今年の音楽ベスト[N]選です。

[N]. [アーティスト名]「[アルバム/曲名]」
[コメント 2〜3文]

[N-1]. [アーティスト名]「[アルバム/曲名]」
[コメント 2〜3文]

[以下同様]

[締め：今年の音楽シーンの総括 2〜3文]
```

---

## 記事作成チェックリスト

- [ ] 書き出しは「〜のライブに行ってきました」系か
- [ ] 主語（私は・僕は）を使っていないか
- [ ] 感嘆符（！）の多用を避けているか
- [ ] 「ヤバい」「神」「最高すぎる」を使っていないか
- [ ] 曲名は「」で囲んでいるか
- [ ] 感情は直接表現せず描写で伝えているか
- [ ] 締めは余韻・期待・感謝のどれかで終わっているか
- [ ] 文体が既存記事と一致しているか（01_記事アーカイブと比較）
